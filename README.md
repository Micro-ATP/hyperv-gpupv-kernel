# hyperv-gpupv-kernel

`hyperv-gpupv-kernel` 是一个 **面向 Hyper-V 虚拟化环境的 Linux 定制内核项目**，核心目标是在 Linux 虚拟机中提供 **GPU-PV（GPU Paravirtualization）** 能力，并尽可能保证内核的稳定性、可维护性与长期可用性。

该项目适用于需要在 Hyper-V 虚拟机中使用宿主机 GPU 资源（图形或计算）的 Linux 使用场景。

---

## 项目背景

Hyper-V 提供了 GPU-PV（GPU 半虚拟化）机制，用于将宿主机 GPU 资源以共享方式提供给虚拟机使用。  
在 Windows 虚拟机中，该机制已经较为成熟，但在 Linux 生态中仍存在以下问题：

- 官方 Linux 内核中对 Hyper-V GPU-PV 的支持较为分散  
- 不同内核版本对 GPU-PV 的实现与行为不一致  
- 发行版默认内核（尤其是 LTS）往往缺失必要补丁或存在兼容性问题  
- 实际使用中容易遇到显示异常、性能受限或稳定性不足的情况  

`hyperv-gpupv-kernel` 的目标，是在 **尽量贴近主线内核设计的前提下**，集中维护并验证 **适用于 Hyper-V GPU-PV 场景的内核实现**，降低使用与维护成本。

---

## 项目目标

本项目并非重新发明内核，而是专注于以下方向：

- 集中维护与 Hyper-V GPU-PV 相关的内核改动
- 在主线或 LTS 内核基础上进行最小侵入式定制
- 提供可复现、可构建、可回退的内核版本
- 尽量避免破坏 Hyper-V 以外场景的正常使用
- 为后续补丁回馈或问题定位提供清晰基线

---

## 适用场景

本内核适用于以下典型使用场景：

- 在 Hyper-V 虚拟机中运行 Linux 桌面环境
- 在 Linux 虚拟机中使用宿主机 GPU 进行图形加速
- 在 Hyper-V 环境下进行 GPU 相关的开发、测试或实验
- 对 GPU-PV 稳定性和行为可控性有要求的用户

---

## 内核策略说明

- 以内核主线或 LTS 版本为基础
- 所有与 GPU-PV 相关的修改尽量模块化、可追踪
- 非必要情况下不引入与 GPU-PV 无关的功能性改动
- 明确区分稳定分支与实验分支，避免混用

---

## 分支规划（示例）

- `main`  
  当前主要维护分支，基于相对稳定的内核版本

- `lts/*`  
  基于 LTS 内核的长期维护分支

- `experimental/*`  
  用于测试新补丁、新实现或上游变更的实验分支

---

## 使用说明

本仓库仅提供内核源码及相关定制内容，不直接提供发行版级安装包。  
用户需根据自身发行版与构建流程自行编译与安装内核。

在使用前，建议具备以下基础：

- Linux 内核编译与安装经验
- 对 Hyper-V 虚拟化环境有基本了解
- 能接受一定程度的实验性风险

---

## 稳定性说明

GPU-PV 在 Linux 上仍属于相对小众且依赖宿主环境的使用场景。  
本项目 **不保证在所有硬件与宿主配置下都能正常工作**，但会尽力保持：

- 内核本身可正常启动
- 不影响非 GPU-PV 场景的基本功能
- 问题可定位、可回退
